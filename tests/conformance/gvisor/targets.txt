# DECLINED: upstream targets deliberately NOT enabled, grouped by why they sit
# outside a single-process syscall translator on the QEMU reference. This block
# is documentation only; "make check-gvisor-targets" reports anything listed
# here (or newly added upstream) as an opt-in candidate.
#
# Networking stack (non-loopback sockets, netlink, iptables, packet, raw,
#   tuntap, *_external): needs a real network stack or external peers.
# Privilege and namespaces (capabilities, seccomp, ptrace, mount, chroot,
#   setns, unshare, pivot_root, cgroup, fuse): needs privileged host setup.
# proc host-specific (proc_*): asserts host-kernel /proc contents.
# pty and devpts (pty_test, pty_root_test, host_pty_test): devpts is broken in
#   the current fixture.
# gVisor-only (kcov_test, getcpu_host_test): gVisor-internal expectations.
# x86-only (32bit_test, arch_prctl_test, sysret_test, vsyscall_test,
#   exceptions_test): not built on AArch64.
